
Ce niveau est similaire au niveau mais avec plus de parametres a prendre en
compte, notamment le masque de sous-réseau.

--------------------------------------------------------------------------------
C’est quoi le masque ?

	C’est une adresse binaire de 32 bits (comme une IP), composée de :
		1 pour les bits appartenant au réseau (partie fixe),
		0 pour les bits appartenant à l’hôte (partie variable).

	Exemple :
	255.255.255.0 → en binaire : 11111111.11111111.11111111.00000000
		Les 24 premiers bits sont le réseau, les 8 derniers sont pour les hôtes.

--------------------------------------------------------------------------------
À quoi sert-il ?

	Déterminer si deux machines sont dans le même réseau :
	En comparant les adresses IP ET les masques, on peut savoir si elles peuvent
	communiquer directement (sans routeur).

	Définir la taille du réseau :
		Plus le masque a de bits à 1, plus le réseau est petit (moins d’hôtes
		possibles).

	Calculer l’adresse du réseau :
		En faisant un ET logique entre l’IP et le masque.

--------------------------------------------------------------------------------
Comment l’écrire ?

	Il existe 3 notations courantes :

		1. Notation décimale pointée (la plus courante)
		Exemple :
			255.255.255.0

		2. Notation CIDR (Classless Inter-Domain Routing) → /24, /30, etc.
		→ Indique le nombre de bits à 1 dans le masque.
		Exemple :
			/30 = 30 bits à 1 → masque 255.255.255.252

		3. Notation binaire (rarement utilisée en pratique)
		Exemple :
			11111111.11111111.11111111.11111100

--------------------------------------------------------------------------------
Est ce que deux machines qui doivent se connecter peuvent avoir un Mask different ?

	-> Non
	Deux machines qui doivent communiquer directement (sans routeur) doivent
	obligatoirement avoir le même masque de sous-réseau.

	Pour une communication directe (sans routeur), les deux machines doivent avoir :
		-> Le même masque,
		-> Et être dans le même réseau (même adresse réseau calculée).

--------------------------------------------------------------------------------
Comment calculer le nombre d'adresse disponibles a partir du Mask de sous-reseau ?

Le masque de sous-réseau permet de calculer le nombre d’adresses IP disponibles
dans un réseau, mais pas toutes sont utilisables pour les machines.

	Masque /24 → bits hôtes = 32 – 24 = 8 bits → 2⁸ = 256 adresses
	Masque /30 → bits hôtes = 2 → 2² = 4 adresses

Dans les adresses disponibles il y en a 2 qui sont deja reservees.
	- Adresse réseau → la première (tous les bits hôtes à 0)
		-> non utilisable pour une machine
	- Adresse de diffusion (broadcast) -> la dernière (tous les bits hôtes à 1)
		-> non utilisable pour une machine

--------------------------------------------------------------------------------
Les adresses 127.x.x.x ne peuvent pas être utilisées pour la communication entre
deux machines sur un réseau privé.

Elles sont réservées par le protocole IPv4 pour le loopback (communication interne
à l’ordinateur lui-même). Aucun trafic réseau ne peut sortir d’une machine vers
une autre en utilisant ces adresses.

1. Le réseau 127.0.0.0/8 (toutes les adresses 127.x.x.x) est réservé au loopback
	RFC 1122 (1989) et RFC 5735 (2010) le confirment.
	Toute adresse dans 127.0.0.0 à 127.255.255.255 est réservée.

	Seule 127.0.0.1 est standardisée comme “localhost”, mais techniquement,
	toutes les adresses du bloc 127/8 peuvent être utilisées localement, mais
	jamais pour la communication réseau entre machines.

2. Autres adresses réservées ou spéciales en IPv4
	Voici les principaux blocs réservés (RFC 5735, RFC 6890) :
	Bloc IP	Usage	Exemple(s)
	0.0.0.0/8	Réseau non spécifié	0.0.0.0
	10.0.0.0/8	Privé (RFC 1918)	10.0.0.1
	127.0.0.0/8	Loopback	127.0.0.1, 127.1.2.3
	169.254.0.0/16	Link-local (APIPA)	169.254.1.1
	172.16.0.0/12	Privé (RFC 1918)	172.16.0.1 à 172.31.255.255
	192.0.0.0/24	IETF protocol assignments	192.0.0.1
	192.0.2.0/24	TEST-NET-1 (documentation)	192.0.2.1
	192.168.0.0/16	Privé (RFC 1918)	192.168.1.1
	198.18.0.0/15	Benchmarking	198.18.1.1
	224.0.0.0/4	Multicast	224.0.0.1
	240.0.0.0/4	Réservé pour usage futur	240.0.0.1
	255.255.255.255	Broadcast limité	—

Methode pour calculer l'IP d'apres le Mask de sous reseau 



	Identifier le réseau :
	Appliquer le masque de sous-réseau (en binaire) à l’adresse IP de référence (ou à n’importe quelle IP du réseau) pour obtenir l’adresse réseau.

	Choisir deux adresses IP valides :
	Dans ce réseau, choisir deux adresses IP différentes, non réservées (ni l’adresse réseau, ni l’adresse de broadcast), et dans la même plage.


Situation :

Tu as une machine avec l’IP 192.168.45.222 et le masque 255.255.255.224 (soit /27).
Tu veux savoir quelle est la plage du réseau dans lequel elle se trouve — et donc quelles autres IPs peuvent lui parler.
Étape 1 : Trouver l’adresse réseau

Le masque 255.255.255.224 = /27 → 27 bits réseau, 5 bits hôtes.

On va appliquer le masque à l’IP 192.168.45.222 pour trouver le réseau.
En binaire :

IP :
192.168.45.222 → 11000000.10101000.00101101.11011110

Masque :
255.255.255.224 → 11111111.11111111.11111111.11100000

→ AND logique :

plain text

11000000.10101000.00101101.11011110
AND
11111111.11111111.11111111.11100000
=
11000000.10101000.00101101.11000000

→ En décimal : 192.168.45.192
Adresse réseau = 192.168.45.192
📏 Étape 2 : Trouver la plage utilisable

    Taille du réseau = 2⁵ = 32 adresses
    Adresse réseau : 192.168.45.192
    Adresse broadcast = 192.168.45.192 + 31 = 192.168.45.223
    Plage utilisable : 192.168.45.193 → 192.168.45.222

    Oui, 192.168.45.222 est bien la dernière IP utilisable dans ce réseau !

🖥️ Donc, pour une autre machine :

Tu peux lui donner n’importe quelle IP entre 192.168.45.193 et 192.168.45.222, sauf 222 si tu veux l’attribuer à la première machine.

Exemple :

    Machine 1 : 192.168.45.222
    Machine 2 : 192.168.45.200
    Masque : 255.255.255.224 pour les deux.

→ Elles sont dans le même réseau → elles peuvent se connecter.
Pourquoi la plage est-elle décalée ?

Parce que le réseau est défini par l’adresse réseau, pas par une IP aléatoire.

→ 192.168.45.222 n’est pas dans le réseau 192.168.45.0/27, mais dans 192.168.45.192/27.

Les réseaux /27 sont découpés tous les 32 adresses :

python

192.168.45.0     → 192.168.45.31
192.168.45.32    → 192.168.45.63
192.168.45.64    → 192.168.45.95
...
192.168.45.192   → 192.168.45.223 ← ton réseau
192.168.45.224   → 192.168.45.255

→ Donc oui, la plage est décalée — elle dépend de l’adresse réseau, pas de l’IP qu’on te donne.
